# exam-rules

Reglas duras del proyecto exam-generator.

- R1: la IA queda fuera del runtime del cliente (solo build-time o servidor).
- R2: cero hex en componentes; solo tokens `--mep-*`.
- R3: UI responsiva 2D.
- R4: solo escritorio (no web publica).
- R5: algoritmo de calificacion REAC bit-exacto.
- R6: modulos desacoplados como paquetes.
- R7: banco de items cifrado.
